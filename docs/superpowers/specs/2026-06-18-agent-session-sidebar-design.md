# Agent 会话层级侧栏设计

**日期：** 2026-06-18
**状态：** 已确认 A 方案
**范围：** 优化左侧会话栏里 Agent 名称和聊天标题的视觉层级，让黑白主题下都能清楚区分“Agent 分组”和“会话条目”。

## 目标

左侧栏要从“按标题分组的列表”升级为“Agent 卡片 + 会话列表”的结构。用户一眼能看出当前浏览的是哪个 Agent、这个 Agent 下有多少会话，以及当前选中的是哪条会话。

## 当前问题

现在 `LeftSidebar.vue` 复用 `vue-element-plus-x` 的 `Conversations` 分组能力，Agent 标题和聊天标题在同一套列表视觉里：

- Agent 名称只是普通 group title，和会话标题界限偏弱。
- 会话数量 badge 比较小，信息存在但不够像 Agent 级别信息。
- 当前会话高亮明显，但当前 Agent 的归属感不够强。
- 分组折叠状态只存在组件内存里，刷新后不保留。

## 设计方案

采用用户确认的 **A. 静稳卡片** 方案：

- 每个 Agent 是一个轻量卡片块。
- Agent 卡片头显示 `Agent 名称 + 会话数`。
- 会话标题作为卡片内的列表项展示。
- 当前 Agent 卡片头和当前会话标题同时高亮。
- 非当前 Agent 的展开/折叠状态记住用户上一次操作。

## 视觉规则

### Agent 卡片

Agent 卡片使用细边框和轻背景，和侧栏背景形成清晰但克制的分区：

- 黑色主题下使用半透明浅色填充和细边框。
- 白色主题下使用白色/浅灰填充和浅边框。
- 当前 Agent 卡片头增加主色侧边线或柔和底色。

卡片之间保留适度间距，避免变成连续列表。

### Agent 头部

Agent 头部包含：

- 展开/收起箭头。
- Agent 名称。
- 会话数 badge。

Agent 名称支持省略号，避免长名称撑破侧栏。会话数 badge 保持小而清楚，不抢当前会话标题的注意力。

### 会话条目

会话标题保持紧凑列表形态：

- 默认态是透明或极轻背景。
- hover 态增加轻背景。
- 当前会话使用主色浅背景和主色文字。
- 当前会话不再和 Agent 头部视觉混在一起。

## 交互规则

- 点击 Agent 头部展开或收起该 Agent。
- 点击“全部展开/全部折叠”继续保留现有功能。
- 点击会话仍然触发原有 `switchSession`。
- 重命名和删除菜单继续使用现有 `Conversations` 内置菜单。
- 折叠状态写入 `localStorage`，key 固定为 `lc-agent:sidebar:collapsed-agent-groups`。
- 如果 Agent 名称不存在或变化，读取折叠状态时自动忽略不存在的旧名称，不做兼容迁移。

## 非目标

- 不替换 `element-plus`。
- 不替换 `vue-element-plus-x`。
- 不重写会话 store、agent store、路由逻辑。
- 不改变会话创建、删除、重命名行为。
- 不兼容历史错误数据。

## 测试策略

新增一个轻量契约测试脚本，检查 `LeftSidebar.vue` 是否具备这次 UI 的关键结构和持久化行为：

- Agent 分组头使用卡片化 class。
- 存在当前 Agent 高亮 class。
- 折叠状态使用固定 localStorage key。
- 移动端仍保留侧栏宽度约束。

继续运行已有前端验证：

```powershell
npm run test:responsive
npm run build
```

## 验收标准

- 左侧栏里 Agent 名称和聊天标题层级清晰。
- Agent 头部显示 Agent 名称和会话数。
- 当前 Agent 和当前会话都能被看出来。
- 用户折叠过的 Agent 分组刷新后仍保持折叠。
- 黑白主题下边框、背景、高亮都有足够对比。
- 手机和电脑端侧栏都没有文字溢出或横向撑宽。
